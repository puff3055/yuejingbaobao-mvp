# 月经宝宝｜项目总入口

这是本项目的唯一项目级入口。任何新负责人、Agent 或协作者，都先读本页，再按任务进入对应文档。

## 当前阶段

> **已有产品愿景与可演示原型；核心用户、第一刚需、Agent 必要性与持续使用动机尚未验证。**

当前工作的重点不是扩展功能或继续包装，而是建立可验证的产品方向。

## 30 秒交接

- 当前定义：月经宝宝是一位拥有个人跨周期记忆，并能检索有来源专业知识与用户授权真实经验的月经专属 Agent。宝宝小窝、知识海、照护礼物、Agent 交友、长期报告、成长与完整生命周期等产品资产均已保存在当前方案中，并按核心、待验证和未来储备分层。
- 当前价值假设：让反复发生的月经状态、照护行动和真实效果形成个人长期经验，减少重复搜索、判断与试错。
- 当前最大不确定性：这个问题是否足够痛、用户是否愿意持续反馈、专属 Agent 是否明显优于搜索/GPT/经期 App。
- 当前 Demo：能展示一次“表达状态—理解—行动—效果反馈—保存/分享”的概念闭环，但不能证明市场需求，也未实现真实跨周期智能。
- 下一项核心决策：公开桌面研究未达到候选门槛；决定是否授权定向招募真实用户，通过最近一次经历回放寻找第一批用户与高代价时刻。

## 文档权威顺序

1. `PROJECT.md`：项目阶段、导航和治理规则。
2. `docs/project/00-老板驾驶舱.md`：当前管理状态与下一项决策。
3. `docs/product/月经宝宝产品方案-v3.0-current.md`：当前产品定义。
4. `docs/research/01-证据台账.md`：外部证据及其强弱。
5. `docs/project/02-产品决策日志.md`：为何做出或撤回某项决定。
6. 其他材料：专项产物或历史背景，不得覆盖以上文件。

研究状态采用两层管理：`docs/research/README.md`提供老板可读的当前入口；各研究目录内的`RUN_STATE.json`提供正在执行批次的机器断点。已经结束的自动化状态文件只能作为历史快照，不再称为“实时状态”。

## 信息状态标签

- **[事实]**：可由实现、正式资料或外部来源直接核验。
- **[研究发现]**：由明确方法和样本支持，同时注明边界。
- **[推断]**：由若干事实推导，仍可能有其他解释。
- **[假设]**：需要验证，不能对外写成产品效果。
- **[决定]**：当前团队主动选择的方向，不等于已获市场证明。
- **[暂缓]**：保留但不占用当前资源。

## 工作规则

- 每项调研必须从一个老板决策开始，保留“决策问题—研究问题—方法—样本—编码—发现—证据强度—产品影响”的链路。
- 新研究只更新证据台账；达到决策门槛后，才更新产品方案和决策日志。
- Demo 文案、Pitch 和海报必须引用当前产品方案，不得用演示数据证明价值。
- 历史文件不删除；冲突内容标为过期或移入归档。
- 产品研究收束不等于删除产品。任何已形成的功能、交互或世界观都应先进入当前方案的产品资产状态表，再决定当前开发、待验证、储备或明确否定。
