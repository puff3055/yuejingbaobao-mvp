# 月经宝宝

> 把月经仪表盘外化为有记忆的 AI 共生伙伴，帮助女性轻松记录身体变化、积累跨周期经验，并为下一次照护与就医沟通提供依据。

[在线体验](https://145438b1.opendeploy.site/)

## 为什么做

月经会反复发生，但很多人的身体经验没有被连续保存：当疼痛、睡眠、情绪或精力发生变化时，她可能不知道该观察什么，记录后也不清楚这些数据有什么用；下一次相似处境仍需重新搜索和试错，就医时也难以完整说明长期、周期性的变化。

月经宝宝尝试把“当时发生了什么—做了什么—真实效果怎样”连接成一份可以回看和纠正的个人周期记忆。它是一位身体潮汐外化出来的共生伙伴，不是备孕工具、医生、诊断系统或通用聊天机器人。

## 核心体验

```text
自然语言描述当前处境
→ 宝宝只补问一个必要问题
→ 用户检查并纠正记录草稿
→ 结合现实限制与个人历史提供一个可选行动
→ 用户反馈真实效果
→ 记录进入个人周期日历
→ 下一次相似处境改变行动排序
→ 整理成可编辑的就医沟通锦囊
```

## 当前实现

- 小窝中的自由文本 Agent 对话与显式联网授权；
- 对用户原话、确认事实和系统推断进行分层；
- 必要风险分流与结构化响应校验；
- 保存前可纠正、保存后仍可编辑或删除的记录；
- “很有帮助 / 有一点帮助 / 没有帮助或更不舒服”效果反馈；
- 负向效果降低或阻止相同行动在类似处境中成为默认建议；
- 个人周期日历、单日多条记录与单指标图层；
- 有来源的周期身体参考图，与个人记录保持边界；
- 根据确认记录生成、复制或打印的就医沟通锦囊；
- 设备本地记录、长期记忆、在线消息和主动分享的分层隐私控制。

## 快速体验

建议使用手机尺寸打开[在线版本](https://145438b1.opendeploy.site/)：

1. 进入“小窝”；
2. 输入：“我现在小腹特别痛，下午还有会”；
3. 回答宝宝提出的一个必要问题；
4. 检查它整理出的记录并选择一个行动；
5. 反馈行动真实效果并确认保存；
6. 进入“周期”，在日历中打开当天记录；
7. 从周期页或“我的”进入就医锦囊。

最终公开前会再次核对线上按钮名称；以实际界面为准。

## 本地运行

环境要求：Node.js 18+、npm。

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 4174
```

打开 `http://127.0.0.1:4174/`。

运行测试和生产构建：

```bash
npm test
npm run build
```

运行带服务端 Agent 接口的版本：

```bash
cp .env.example .env
# 在 .env 中配置自己的兼容服务，真实密钥不要提交到 Git
npm start
```

## 环境变量

| 变量 | 用途 |
|---|---|
| `AGENT_API_BASE_URL` | OpenAI 兼容 API 的基础地址 |
| `AGENT_API_MODEL` | Agent 使用的模型名称 |
| `AGENT_API_KEY` | 服务端密钥，只能放在本地或部署平台的秘密存储中 |
| `AGENT_SEARCH_URL` | 可选的检索服务地址 |
| `NCBI_TOOL` | 可选的 PubMed 工具标识 |
| `NCBI_EMAIL` | 可选的 PubMed 联系邮箱 |
| `NCBI_API_KEY` | 可选的 PubMed API Key |

浏览器端不会读取 `AGENT_API_KEY`。

## 边界与隐私

- 这是比赛原型，不是医疗器械，不诊断、开药或确认排卵与激素状态；
- 日期与周期位置只能作为大概参考，不能自动生成个人身体事实；
- 只有用户确认的内容和效果才进入个人记忆；
- 演示记录、社区内容和专业知识必须与真实个人事实分开；
- 专业内容仍需要持续进行妇产科审核；
- 真实健康数据和 API 密钥不应提交到公开仓库。
